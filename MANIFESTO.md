# MANIFESTO — B.E.N.
### *Bridge Engine for Native CFML*

---

## O problema que ninguém quis resolver

CFML roda em silêncio.

Roda em bancos, em hospitais, em sistemas de governo, em plataformas de e-commerce que processam milhões de transações por dia. Roda em empresas que existem há décadas e continuarão existindo por décadas mais. Roda onde a estabilidade importa mais que a novidade.

E enquanto isso, o mundo do tooling seguiu em frente sem olhar para trás.

Os grandes frameworks de IA ignoraram CFML. Os servidores de linguagem modernos ignoraram CFML. O ecossistema MCP — que está redefinindo como desenvolvedores interagem com agentes de IA — foi construído sem uma única implementação nativa para CFML.

A comunidade é pequena. Leal. Experiente. E completamente órfã de ferramentas modernas.

Isso não é acidente. É negligência de mercado. E negligência de mercado é oportunidade.

---

## O que estamos construindo

**B.E.N.** é uma ponte.

Entre o ecossistema CFML — maduro, estável, subestimado — e a nova geração de agentes de IA, IDEs inteligentes e fluxos de trabalho orientados a modelos de linguagem.

É o primeiro MCP Server nativo para CFML. É um agente autônomo com roteamento inteligente entre múltiplos LLMs. É um sistema de gerenciamento de chaves que torna o acesso a IA acessível mesmo para quem opera com orçamento limitado.

Mas mais do que uma ferramenta, B.E.N. é uma declaração: **desenvolvedores CFML merecem tooling à altura do que constroem.**

## O que é gratuito e o que é comercial

Acreditamos que ferramentas que servem a comunidade devem ser acessíveis à comunidade.

**É gratuito para sempre:**
- Uso pessoal
- Projetos open source
- Aprendizado, experimentação e contribuição

**Requer licença comercial:**
- Uso em produtos ou serviços que geram receita
- Integração em stacks corporativos
- Redistribuição como parte de soluções pagas

Este modelo não é uma barreira. É o que torna possível continuar construindo.

Em 1º de março de 2036, todo o código se torna Apache 2.0 — aberto, irrestrito, permanente.

## Sobre o nome

**B.E.N.** — *Bridge Engine for Native CFML* — é uma homenagem em vida a **Ben Forta**.

Poucos na história de qualquer linguagem de programação fizeram tanto por uma comunidade com tanta consistência e tanto cuidado. Ben documentou, ensinou, evangelizou e humanizou o CFML por décadas. Sem ele, a comunidade seria uma fração do que é.

Este projeto existe em parte por causa do que ele construiu. Nomear a ferramenta foi a forma mais direta de dizer isso em voz alta.
